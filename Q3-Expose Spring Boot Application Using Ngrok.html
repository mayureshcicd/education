How to Expose Spring Boot Application Using Ngrok

Ngrok allows you to expose a local server (like your Spring Boot application) to the internet via a secure public URL. This is especially useful for testing webhooks, APIs, or sharing your local app with others.

Step 1: Download Ngrok

  1. Go to the official Ngrok website: https://ngrok.com/download
  2. Choose your operating system (Windows, Linux, Mac, etc.)
  3. Download the zip file

Step 2: Install Ngrok

After downloading:

  • Extract the downloaded zip file
  • You will get an executable file named ngrok (or ngrok.exe on Windows)
  • Move it to a known location like /usr/local/bin (Linux/Mac) or keep it in a folder on Windows and add that folder to your PATH

Step 3: Sign Up & Get Your Authtoken

  1. Create a free account on https://dashboard.ngrok.com/signup
  2. Once logged in, go to the dashboard
  3. Copy your unique AuthToken from the "Setup & Installation" section

Step 4: Connect Ngrok with Your Account

Open a terminal and run:

ngrok config add-authtoken <YOUR_AUTHTOKEN>

Example:

ngrok config add-authtoken 2MkYkFjB1uSxEXAMPLE1v8czp1Zd3k6U8abB

Step 5: Start Your Spring Boot Application

Make sure your Spring Boot app is running. Typically, it's accessible at:

http://localhost:8080

You can run your Spring Boot app using:

mvn spring-boot:run
or
java -jar target/your-app.jar

Just replace 8080 with your actual port. For example:

  • If your app runs on localhost:9090, run:
    • ngrok http 9090
    This tells Ngrok to expose your local server running on port 9090.
  • If it runs on localhost:8080, run:
    • ngrok http 8080
     This tells Ngrok to expose your local server running on port 8080.

Step 7: Access Your Public URL

After running the above command, you will see output like this:

Version                       3.9.0
Forwarding                    http://abc123.ngrok.io -> http://localhost:8080
Forwarding                    https://abc123.ngrok.io -> http://localhost:8080

You can now access your Spring Boot app using the https://abc123.ngrok.io public URL from anywhere.

Step 8: Use Cases

  • API testing from external tools like Postman
  • Webhook testing for services like Stripe, Razorpay, PayPal, etc.
  • Share your app with teammates or clients without deploying

Note:

  • Each time you restart Ngrok, the public URL will change (unless you are on a paid plan with reserved domains)
  • To make it permanent or use subdomains, upgrade to a paid plan on Ngrok

Troubleshooting Tips

  • Make sure your Spring Boot app is running and accessible at localhost:8080
  • Ensure Ngrok is properly authenticated with your token
  • If you see errors, check Ngrok logs and Spring Boot logs for port issues

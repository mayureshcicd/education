Core MongoDB Concepts

Document

The basic unit of data in MongoDB, stored in BSON format (Binary JSON). Documents are analogous to rows in relational databases but with a flexible schema.

Example: A user document containing id, name, email, and address fields.
Spring Boot Annotation: @Document
@Document(collection = "users")
public class User {
  @Id
  private String id;
  private String name;
  private String email;
}

Collection

A grouping of MongoDB documents. Collections are analogous to tables in relational databases but don't enforce a schema.

Example: A "users" collection containing all user documents.
Usage: Documents within a collection can have different fields, providing schema flexibility.

Database

A physical container for collections. Each database gets its own set of files on the file system.

Example: An "ecommerce" database containing collections for users, products, and orders.
Spring Boot Configuration:
spring.data.mongodb.database=myapp

Indexing & Performance - Q&A

7
What is indexing in MongoDB?
Answer: Indexing is a technique to improve query performance by creating special data structures that allow faster data retrieval.
Simple Analogy: Like an index in a book that helps you find information quickly without reading every page.
8
What are the different types of Indexing in MongoDB?
Answer: MongoDB supports several index types:
  • Single Field: Index on one field
  • Compound: Index on multiple fields
  • Multikey: Index on array fields
  • Text: For text search
  • Geospatial: For location data
  • Hashed: For sharding
  • TTL: Auto-delete after time
13
What is the difference between primary key and secondary key in MongoDB?
Answer:
  • Primary Key (_id): Unique identifier for each document, automatically indexed
  • Secondary Key: Any other field that is indexed to improve query performance
// Primary key (automatically indexed)
@Id
private String id;

// Secondary key (manually indexed)
@Indexed
private String email;

Spring Data MongoDB Components

MongoRepository

A Spring Data interface that provides CRUD operations and query methods for MongoDB documents without implementation.

Usage: Extend this interface to automatically get common database operations.
public interface UserRepository extends MongoRepository<User, String> {
  List<User> findByName(String name);
  List<User> findByEmailAndActiveTrue(String email);
}

MongoTemplate

The central class in Spring Data MongoDB for executing database operations, providing a rich feature set for interacting with MongoDB.

Usage: Used for complex queries, aggregations, and operations not covered by repository interfaces.
@Autowired
private MongoTemplate mongoTemplate;

Query query = new Query();
query.addCriteria(Criteria.where("age").gt(18));
List<User> users = mongoTemplate.find(query, User.class);

@Query Annotation

Annotation used to declare finder queries directly on repository methods, using MongoDB JSON query syntax.

Usage: For custom queries that can't be expressed using the method name convention.
@Query("{ 'age' : { $gt: ?0, $lt: ?1 } }")
List<User> findUsersByAgeBetween(int minAge, int maxAge);

Data Operations - Q&A

9
What is mapReduce in MongoDB?
Answer: MapReduce is a programming model for processing large data sets with a parallel, distributed algorithm on a cluster.
Simple Explanation: Like having multiple workers process different parts of a big task and then combining their results.
10
How to store images, pdf in MongoDB using GridFS?
Answer: GridFS is a specification for storing and retrieving large files (over 16MB) by dividing them into smaller chunks.
// Store file using GridFS
GridFSBucket gridFSBucket = GridFSBuckets.create(database);
ObjectId fileId = gridFSBucket.uploadFromStream("image.jpg", inputStream);

// Retrieve file
gridFSBucket.downloadToStream(fileId, outputStream);
11
What is the aggregation pipeline in MongoDB?
Answer: Aggregation pipeline is a framework for data aggregation that processes documents through a series of stages.
Simple Analogy: Like an assembly line where raw data goes through different processing stations to become useful information.
// Example aggregation pipeline
Aggregation aggregation = Aggregation.newAggregation(
  Aggregation.match(Criteria.where("status").is("active")),
  Aggregation.group("department").count().as("count"),
  Aggregation.sort(Sort.Direction.DESC, "count")
);
12
What is the difference between update & Save in MongoDB?
Answer:
  • Save: Replaces entire document if _id exists, inserts new document if _id doesn't exist
  • Update: Modifies specific fields without replacing the entire document
// Save - replaces entire document
userRepository.save(user);

// Update - modifies specific fields
Query query = new Query(Criteria.where("id").is(userId));
Update update = new Update().set("email", newEmail);
mongoTemplate.updateFirst(query, update, User.class);

MongoDB Annotations in Spring Boot

@Id Annotation

Marks a field as the primary identifier for the document. If no value is provided, MongoDB will generate an ObjectId.

@Id
private String id;

@Field Annotation

Allows customizing the name of the field as it will be stored in the MongoDB document.

@Field("user_name")
private String name;

@Indexed Annotation

Declares an index on the field for faster query performance. Can be configured for unique, sparse, or TTL indexes.

@Indexed(unique = true)
private String email;

Advanced Features - Q&A

14
How does MongoDB ensure data consistency?
Answer: MongoDB ensures data consistency through:
  • Write Concern: Controls acknowledgment level for write operations
  • Read Concern: Controls isolation level for read operations
  • Replica Sets: Maintain multiple copies of data
  • Transactions: Multi-document ACID transactions (v4.0+)
15
How does MongoDB handle schema changes?
Answer: MongoDB handles schema changes flexibly through:
  • Schema-less Design: Documents in same collection can have different fields
  • Backward Compatibility: Old applications can read new documents (ignore unknown fields)
  • Migration Scripts: Update existing documents when needed
16
How does MongoDB handle transactions?
Answer: MongoDB supports multi-document ACID transactions (since v4.0) that can span multiple operations, collections, and databases.
@Transactional
public void transferMoney(String from, String to, BigDecimal amount) {
  // Multiple database operations in single transaction
  accountRepository.debitAccount(from, amount);
  accountRepository.creditAccount(to, amount);
  // Both succeed or both fail
}
19
How does MongoDB handle security?
Answer: MongoDB provides multiple security features:
  • Authentication: Username/password, LDAP, Kerberos
  • Authorization: Role-based access control
  • Encryption: TLS/SSL for network, encryption at rest
  • Auditing: Track database operations
20
What are the different data models in MongoDB? (embedded & reference)
Answer: MongoDB supports two main data modeling approaches:
  • Embedded Documents: Related data stored within a single document (one-to-few relationships)
  • Document References: Related data stored in separate documents with references (one-to-many relationships)
// Embedded document (one-to-few)
public class User {
  private List<Address> addresses; // Embedded
}

// Document reference (one-to-many)
public class Order {
  @DBRef
  private User user; // Reference
}
21
What is the difference between join & lookup in MongoDB?
Answer:
  • Join (SQL): Combines rows from multiple tables at query time
  • $lookup (MongoDB): Performs left outer join between collections in aggregation pipeline
// $lookup example in aggregation
Aggregation.newAggregation(
  Aggregation.lookup("orders", "userId", "userId", "userOrders")
);
22
How does MongoDB handle data backup and recovery?
Answer: MongoDB provides several backup options:
  • mongodump/mongorestore: Binary export/import of database
  • mongoexport/mongoimport: JSON/CSV export/import
  • File System Snapshots: Point-in-time backups
  • Cloud Manager: Automated backup service
  • Replica Sets: Built-in redundancy
